webpack系列-3搭建webpack环境

发表于 2019-08-28

基于nodejs的，webpack正确安装方式
-

  • npm install webpack webpack-cli -g
    全局安装webpack
  • 项目内安装
    npm install webpack webpack-cli –save-dev

1webpack.config.js基础配置

module.exports = {
    mode: 'development', //production编译出代码会被压缩，development不会压缩
    entry: {
        main: './src/index.js'
    },
    output: {
        filename: 'bundle.js',
        path: path.resolve(__dirname, 'dist')
    }
}

直接运行 npm webpack –config webpack.config.js，以webpack.config.js为配置文件进行打包

脚本文件的方式
"dev":"webpack --mode development",
"prod":"webpack --mode production"

2 webpack.config.js实践

1.什么是loader，什么是plugin

loader用于加载某些资源文件。因为webpack本身只能打包common.js规范的js文件，对于其他资源如css，img等，是没有办法加载的，这时就需要对应的loader将资源转化，从而进行加载。

plugin用于扩展webpack的功能。不同于loader，plugin的功能更加丰富，比如压缩打包，优化，不只局限于资源的加载。

文件
url-loader 像 file loader 一样工作，但如果文件小于限制，可以返回 data URL
file-loader 将文件发送到输出文件夹，并返回（相对）URL


const path = require('path');    
module.exports = {
    mode: 'development',
    entry: {
        main: './src/index.js'
    },
    module: {
        rules: [{
            test: /\.(jpg|png|gif)$/,
            use: {
                loader: 'url-loader',  //完全可以替换file-loader，去掉limit
                options: {
                    name: '[name]_[hash].[ext]',// 单引号，告诉是字符串，占位符
                    outputPath: 'images/',
                    limit: 10240
                }
            } 
        }]
    },
    output: {
        filename: 'bundle.js',
        path: path.resolve(__dirname, 'dist')
    }
}

name: ‘[name]_[hash].[ext]’,// 单引号，告诉是字符串，占位符官网解释

https://www.webpackjs.com/loaders/file-loader/#%E9%80%89%E9%A1%B9

name是基本文件名字 hash是随机hash ext是文件后缀
