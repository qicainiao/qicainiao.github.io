webpack系列-4使用loader打包静态文件－图片

发表于 2019-08-28

我们图片打包好的名字变成了很长的字符串，如果想不变动名字怎么办。现在的名字是avatar.jpg，打包后的名字我希望也是avatar.jpg。这个时候我们需要去改配置，在使用file-loader的时候，我们可以额外的加个参数，options，再把配置放进去。

file-loader

基础配置

placeholder

这个时候打包出来的文件就不会变名字，这种配置的语法，我们把他叫做placeholder，中文就是占位符，placeholder还有其它的一些配置，比如加上hash

更多文件类型

这个时候，原始文件名后面就会跟上hash。除了jpg，还可以打包png,gif等。现在图片是打包在根目录下， 我希望打包到一个imgs的文件夹里，那怎么办？

url-loader

除了file-loader，还有一种url-loader。他可以完成file-loader的一切功能

基础配置

我们把file-loader换成url-loader打包，完全没有问题，但是我们发现图片没有被打包到dist目录下，我们运行index.html的时候，没问题，这很奇怪，为什么呢？

我们打开bundle.js发现图片被打包成了base64的格式。所以url-loader是这么一个loader。这样打包的好处是省了一次请求，但是如果这个图片很大，加载js的时间也会很长，页面不会立马显示出来，所以url-loader最佳的使用方式是什么？

如果图片是1,2kb，用url-loader很好，但是图片很大，就还是使用file-loader。要想实现这种最佳实践怎么办？

参数配置

加了一个limit，如果小于2个字节，就打包成base64，如果大于2个字节，就还是使用file-loader打包。

所以url-loader多了limit的配置项，大于2048的时候会跟file-loader的功能一模一样

1webpack.config.js基础配置

改到这，我们在打包到过程中，经常会用npx这样到一个命令，但是如果经常写react,vue，可能从来没有使用过这个命令。而是使用npm run这样的命令。所以这就是运用npm script简化我们的打包代码

我们打开package.json这个文件

这个文件里面又一个scripts标签配置项。这是一个对象，我们可以在这里配置一些内容

比如这样，这个时候我们不使用npx webpack了。直接运行
npm run bundle
一样可以帮我们打包。所以npm run bundle执行的时候就是执行script的bundle命令。这个bundle命令底层在帮你干什么，就是帮你执行webpack这个命令，那么又有一个问题。直接写webpack是在全局里面去找webpack。是不是应该加个npx呢？实际上在scripts里面使用webpack,他会先到工程目录下的node_modules里面去找是否安装了webpack这个指令。现在去写，直接npm run bundle就可以了。

这个时候把index.html移到dist下面，就可以直接运行起来了

2 webpack.config.js实践

什么是loader，什么是plugin

loader用于加载某些资源文件。因为webpack本身只能打包common.js规范的js文件，对于其他资源如css，img等，是没有办法加载的，这时就需要对应的loader将资源转化，从而进行加载。

plugin用于扩展webpack的功能。不同于loader，plugin的功能更加丰富，比如压缩打包，优化，不只局限于资源的加载。

文件
url-loader 像 file loader 一样工作，但如果文件小于限制，可以返回 data URL
file-loader 将文件发送到输出文件夹，并返回（相对）URL


const path = require('path');    
module.exports = {
    mode: 'development',
    entry: {
        main: './src/index.js'
    },
    module: {
        rules: [{
            test: /\.(jpg|png|gif)$/,
            use: {
                loader: 'url-loader',  //完全可以替换file-loader，去掉limit
                options: {
                    name: '[name]_[hash].[ext]',// 单引号，告诉是字符串，占位符
                    outputPath: 'images/',
                    limit: 10240
                }
            } 
        }]
    },
    output: {
        filename: 'bundle.js',
        path: path.resolve(__dirname, 'dist')
    }
}

name: ‘[name]_[hash].[ext]’,// 单引号，告诉是字符串，占位符官网解释

https://www.webpackjs.com/loaders/file-loader/#%E9%80%89%E9%A1%B9

name是基本文件名字 hash是随机hash ext是文件后缀

webpack系列-1 webpack究竟是什么

发表于 2019-08-28

1 webpack究竟是什么？

以前写html,js的写法如下：

index.html文件

<!DOCTYPE html>
 <html lang="en">
 <head>
     <meta charset="UTF-8">
     <title>这是最原始的网页开发</title>
 </head>
 <body>
     <p>这是我们的网页内容</p>
     <div id='root'></div>
     <script src='./index.js'></script>
 </body>
 </html>

index.js文件

var dom = document.getElementById('root');
var header = document.createElement('div');
header.innerText = 'header';
dom.append(header);

var content = document.createElement('div');
content.innerText = 'content';
dom.append(content);

var sidebar = document.createElement('div');
sidebar.innerText = 'sidebar';
dom.append(sidebar);

随着业务增加，如果用这种面向过程的编程方式代码就会很长，所有的js逻辑都堆在一个文件，最后难以维护。
采用面向对象的编程方式

<!DOCTYPE html>
 <html lang="en">
 <head>
     <meta charset="UTF-8">
     <title>这是最原始的网页开发</title>
 </head>
 <body>
     <p>这是我们的网页内容</p>
     <div id='root'></div>
     <script src='./index.js'></script>
 </body>
 </html>

content.js文件

function Content() {
    var dom = document.getElementById('root');
    var content = document.createElement('div');
    content.innerText = 'content';
    dom.append(content);
}

module.exports = Content;

header.js文件

    function Header() {
    var dom = document.getElementById('root');
    var header = document.createElement('div');
    header.innerText = 'header';
    dom.append(header);
}

module.exports = Header;

sidebar.js

function Sidebar() {
    var dom = document.getElementById('root');
    var sidebar = document.createElement('div');
    sidebar.innerText = 'sidebar';
    dom.append(sidebar);
}

module.exports = Sidebar;

index.js

var Header = require('./header.js');
var Sidebar = require('./sidebar.js');
var Content = require('./content.js');

new Header();
new Sidebar();
new Content();

这个文件各司其职，便于维护管理。
不存在依赖颠倒出现的问题
运行上面代码
npm init

package.json文件 

"devDependencies": {
 "webpack-cli": "^3.1.2"
  },
 "dependencies": {
   "webpack": "^4.25.1"
 }

执行
npx webpack index.js
